### Q16. 批注显示方式

**天道/刘和平等 Agent 在章节中添加的批注如何显示？**

**方案 A - 侧边栏批注**（类似 Word）:
```
┌──────────────────────────────┬────────────┐
│ 李明站在青云宗的山门前，      │ [天道]     │
│ 望着云雾缭绕的山峰...        │ 建议增加   │
│                              │ 环境描写   │
│ ↑ 批注位置标记               │ 👍 👎 [×]  │
├──────────────────────────────┴────────────┤
```

**方案 B - 内联批注**:
```
李明站在青云宗的山门前，望着云雾缭绕的山峰...

┌─────────────────────────────────────┐
│ 💬 天道: 建议在此增加环境描写，     │
│        突出青云宗的仙气氛围         │
│        [接受] [修改] [忽略]         │
└─────────────────────────────────────┘

...
```

**方案 C - 悬浮批注**:
```
李明站在青云宗的山门前，望着云雾缭绕的山峰...
│
└─ 鼠标悬停显示: [天道] 建议增加环境描写
```

**你的选择是？**以上三种均不符合AI CODING的工作流程和思路格式，我们可以这样设定：对于文本，一般ai coding最容易支持的是markdown格式，哪怕用户最终想导出或者下载，也完全可以用一个脚本快速转为word或者其他常见的文档格式，然后给出下载链接
 通用方法：HTML 注释（推荐）
这是最安全的方式，几乎在所有 Markdown 编辑器和支持 HTML 的渲染器中都能生效。注释内容在渲染后的页面上不可见，但在源代码中可见。
语法：

<!-- 这是一个注释 -->
<!-- 
  这也是一个注释
  可以跨越多行
-->
示例：

# 标题

<!-- 这是一段不会显示在最终页面上的说明文字 -->

这里是正文内容。

<!-- 
多行注释示例：
1. 第一点
2. 第二点
-->
