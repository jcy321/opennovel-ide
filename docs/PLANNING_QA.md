## 📋 规划原则

1. **完全删除现有前端** (SvelteKit)，以 VS Code 为基础重建

## ❓ 第一批问题：核心架构

### Q1. 客户端形式

VS Code 有两种部署形式：
- **桌面应用** (Electron): 下载安装包，本地运行
- **Web 版** (code-server/VSCodium): 浏览器访问

**你的偏好是？**
- [ ] A. 仅桌面应用（类似 VS Code 本体）
- [ ] B. 仅 Web 版（类似 code-server）
- [ √] C. 两者都要（桌面 + Web 双版本）
我的想法说明：选择双版本是为了后续此项目具备商业价值，2026年年初openclaw快速火爆全球，但是其部署难度对于一般用户还是极高，虽然多个LLM供应商都察觉到背后蕴含的庞大tokens消耗量意味着商机，但是铺开的困难还是限制了其商业价值，我想提前落子布局。也就是说opennovel需要有三种使用端模式，比如使用opencode只是要在部署机上输入opencode即可，即在后端opencore不变的前提下，opennovel需要有不同的参数决定不同的使用方式，比如如果用户在部署机终端输入opennovel -web启动，则opennovel则以web模式启动，允许用户使用web版，即浏览器访问http://服务器地址：端口连接到opennovel实例；也允许用户部署后，opennovel还需要提供一个vscode插件，使用原版的vscode安装该插件，简单的填入服务器地址和opennovel监听的ide端口，即可连接到opennovel，此模式需要用户在部署机上输入opennovel -ext启动，如此opennovel启动后监听ext端口；最理想的是用户必须下载我们此次开发的opennovel-ide，启动后弹窗要求用户输入服务器地址和端口，opennovel默认启动也是监听ide端口，即6688,即如果只输入opennovel即以此模式启动。
